# OSAI Kit — Architecture

## Overview

OSAI Kit is a terminal UI (TUI) application built with [Ink](https://github.com/vadimdemedes/ink) + React that analyzes a codebase, recommends the best open-source LLM for the stack, and optionally deploys it locally via Ollama.

It runs entirely serverless — no API keys, no backend — everything happens on the user's machine.

## Entry points and commands

### `osaikit` (no args) — Interactive wizard
- Renders `Wizard` component (6 questions)
- Calls `recommend()` → `fetchAllLeaderboards()`
- Shows `Results` screen

### `osaikit --repo <path-or-url>` — Auto-detect mode
- `scanRepo()` analyzes the local filesystem (or cloned remote)
- Passes detected inputs directly to recommendation engine
- Skips wizard, shows results immediately

### `osaikit run local --repo <path>`
- Same analysis as above
- Then runs Ollama detection → model pull → API verification

### `osaikit run local --model <id>`
- Skips analysis, uses specified model directly
- Runs Ollama flow

### `osaikit refresh`
- Clears in-memory caches from all API modules
- Calls `fetchAllLeaderboards()` again
- Writes updated `leaderboard-cache.json`

### `osaikit list [--family] [--on-device] [--legacy] [--json]`
- Filters the `MODELS` array and prints a table or JSON

## Development workflow

### Run with repo analysis (local path)

```bash
npm run dev -- --repo .
```

The `--` separator tells npm to pass remaining arguments to the script. They get appended to the end of `node build.js && node dist/cli.js`, so `--repo .` reaches the CLI.

### Run with repo analysis (remote URL)

```bash
npm run dev -- --repo https://github.com/user/repo
```

The analyzer detects the URL, clones the repo to a temp directory, analyzes it, and cleans up.

### Shortcut script

```bash
npm run dev:repo -- <path-or-url>
```

Equivalent to `npm run dev -- --repo <path-or-url>` but shorter and self-documenting.

### Run tests

```bash
npm test
```

Uses Node.js built-in test runner (`node --test`). Tests are colocated with source files as `*.test.js`.

## Key design decisions

1. **Zero npm dependencies for HTTP** — Uses Node.js built-in `fetch` (available since Node 18).
2. **esbuild for bundling** — Fast, single-file output, inline sourcemaps.
3. **Ink + React for TUI** — Component-based rendering with automatic diffing.
4. **No config files** — Everything is auto-detected or asked interactively.
5. **Offline-first** — Leaderboard data ships as a bundled cache; refresh is opt-in.

## Extending

- **Add a new command** → Add handler in `src/cli.js` `COMMANDS` map, create file in `src/commands/`.
- **Add a new API source** → Create file in `src/api/`, implement fetch + parse, wire in `src/api/index.js`.
- **Add a new model** → Add entry to `src/engine/models.js`.
- **Add a new framework detector** → Add entry to `FRAMEWORK_DETECTORS` array in `src/analyzer/repo.js`.
